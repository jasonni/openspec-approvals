## ADDED Requirements

### Requirement: 儀表板總覽 MUST 呈現專案健康度指標
系統 SHALL 為所選 `projectId` 提供儀表板總覽，包含進行中變更/規格數量、任務總數、完成百分比與近期活動。

#### Scenario: 有效專案可顯示總覽
- **WHEN** 使用者以有效的 `projectId` 開啟儀表板
- **THEN** 系統 MUST 顯示由目前 OpenSpec 變更 artifacts 與核准/活動資料計算出的總覽指標

#### Scenario: 專案資料為空時可正常處理
- **WHEN** 專案沒有任何進行中的變更
- **THEN** 系統 MUST 顯示零狀態指標與建立或選擇變更的引導資訊

### Requirement: 儀表板 MUST 提供依文件類型的規格詳情導覽
系統 SHALL 提供規格詳情導覽，允許使用者透過直接 URL 開啟某個變更的 requirements、design 與 tasks 視圖。

#### Scenario: 直接連結可開啟指定文件視圖
- **WHEN** 使用者直接導向某一文件類型的規格詳情 URL
- **THEN** 系統 MUST 載入指定變更並將對應文件視圖設為啟用狀態

#### Scenario: 無效文件視圖可安全回退
- **WHEN** 文件導向路由引用了不支援的視圖
- **THEN** 系統 MUST 回退到有效的預設文件視圖，且頁面不得中斷

### Requirement: 任務管理 MUST 支援階層式進度視覺化
系統 SHALL 將任務內容解析為階層模型，並顯示整體與分段層級的進度指標。

#### Scenario: 可呈現階層式任務
- **WHEN** 任務內容包含編號階層項目（例如 `1.0`、`1.1`、`1.1.1`）
- **THEN** 系統 MUST 依階層順序呈現任務並保留父子關係

#### Scenario: 任務狀態變更後更新進度
- **WHEN** 任務狀態改為完成或未完成
- **THEN** 系統 MUST 重新計算並更新儀表板中的整體與分段進度數值

### Requirement: 儀表板 MUST 顯示即時連線與更新狀態
系統 SHALL 對使用者揭露即時狀態，包含連線狀態，以及與索引、核准、規格/任務變更相關的更新通知。

#### Scenario: 使用者可見連線狀態
- **WHEN** 即時串流發生連線、重連或斷線
- **THEN** 系統 MUST 在儀表板介面顯示對應的連線狀態指示

#### Scenario: 出現即時更新通知
- **WHEN** 系統收到與目前專案相關的更新事件
- **THEN** 系統 MUST 顯示使用者可見通知，說明更新類型與目標變更/情境

### Requirement: 儀表板 MUST 支援鍵盤可及導覽
系統 SHALL 為主要儀表板流程提供可由鍵盤操作的導覽與語意化可及性支援。

#### Scenario: 鍵盤快捷鍵切換作用中視圖
- **WHEN** 使用者觸發支援的儀表板鍵盤快捷鍵
- **THEN** 系統 MUST 移動焦點與/或啟用對應儀表板區段，且不需指標裝置操作

#### Scenario: 提供螢幕閱讀器語意資訊
- **WHEN** 輔助科技讀取儀表板介面
- **THEN** 系統 MUST 為導覽、分頁、狀態指示與可操作控制項提供語意標籤/角色

### Requirement: 儀表板 MUST 支援持久化顯示偏好設定
系統 SHALL 支援儀表板呈現的持久化偏好設定，包含主題與任務列表顯示選項。

#### Scenario: 主題偏好可跨工作階段保留
- **WHEN** 使用者變更儀表板主題
- **THEN** 系統 MUST 在後續造訪時保留並重新套用該偏好

#### Scenario: 任務可見性偏好可還原
- **WHEN** 使用者切換任務顯示選項（例如顯示/隱藏已完成項目）
- **THEN** 系統 MUST 在再次開啟儀表板時還原該偏好設定
